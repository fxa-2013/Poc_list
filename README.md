# 漏洞管理平台

一个基于Python Flask开发的漏洞管理平台，支持导入Excel格式的漏洞扫描报告，智能关联资产信息，实现漏洞的可视化管理。

## 功能特点

### 1. Excel报告导入
- 支持多种漏扫平台导出的Excel格式报告
- 智能识别不同格式的列名（中文/英文）
- 可按风险等级筛选导入（如仅导入高危漏洞）
<img width="1719" height="798" alt="image" src="https://github.com/user-attachments/assets/5cd7a4e3-7c5d-4e58-990b-c8673404159d" />

### 2. 资产管理
- 支持添加、编辑、删除资产信息
- 资产信息包括：IP地址、主机名、资产名称、类型、所属部门、运维人员、联系电话、厂商等
- 资产与漏洞自动关联
<img width="1712" height="797" alt="image" src="https://github.com/user-attachments/assets/78ff9825-4ec3-43ad-9954-2883a974af96" />

### 3. 漏洞管理
- 漏洞列表展示与搜索
- 按风险等级、状态筛选漏洞
- 漏洞状态跟踪（未修复、修复中、已修复、忽略）
- 漏洞详情查看

### 4. 数据可视化
- 仪表盘展示统计数据
- 风险等级分布图表
- 漏洞状态分布

## 技术栈

- **后端**：Python 3.8+, Flask, SQLAlchemy
- **前端**：HTML5, Bootstrap 5, JavaScript
- **数据库**：SQLite（支持扩展到MySQL等）
- **依赖**：pandas, openpyxl

## 安装部署

### 方法一：在线安装（有网络环境）

1. 克隆或下载项目到本地
2. 安装依赖
   ```bash
   pip install -r requirements.txt
   ```
3. 启动应用
   ```bash
   python app.py
   ```
4. 访问 http://localhost:5000

### 方法二：离线安装（无网络环境）

1. **在有网络的环境中**：
   - 下载项目代码
   - 运行依赖下载脚本
     ```bash
     python download_deps.py
     ```
   - 等待依赖包下载完成

2. **在离线环境中**：
   - 将整个项目目录复制到离线服务器
   - 运行离线安装脚本
     ```bash
     python install_offline.py
     ```
   - 启动应用
     ```bash
     python app.py
     ```
   - 访问 http://localhost:5000

## 使用指南

### 1. 首次使用
- 进入系统后，先添加资产信息（资产管理 -> 添加资产）
- 填写资产的IP地址、运维人员、联系电话等信息

### 2. 导入漏洞报告
- 进入 "导入报告" 页面
- 选择要导入的Excel文件
- 选择报告来源（可选）
- 选择风险等级筛选（可选，如仅导入高危漏洞）
- 点击 "开始导入"

### 3. 查看漏洞
- 进入 "漏洞管理" 页面
- 查看所有导入的漏洞
- 使用筛选功能快速定位特定漏洞
- 点击漏洞名称查看详情
- 更新漏洞状态

### 4. 资产管理
- 进入 "资产管理" 页面
- 查看、编辑、删除资产信息
- 点击资产的IP地址查看该资产的漏洞列表

## 支持的Excel格式

系统支持以下列名的Excel文件：

| 字段 | 支持的列名 |
|------|------------|
| 漏洞名称 | 漏洞名称、漏洞、名称、Vulnerability |
| 风险等级 | 风险等级、等级、严重程度、Severity、Risk |
| IP地址 | IP地址、IP、目标、Target、Host |
| 其他可选 | 漏洞类型、CVE编号、描述、修复建议、端口、协议 |

## 注意事项

1. Excel文件大小建议不超过10MB
2. 首次运行时会自动创建SQLite数据库文件
3. 为保证数据安全，建议定期备份数据库文件
4. 支持的Excel格式为.xlsx和.xls

## 系统默认配置

- 默认监听地址：0.0.0.0:5000
- 默认数据库：sqlite:///vuln_report.db
- 文件上传限制：16MB

## 故障排查

### 常见问题

1. **Excel文件导入失败**
   - 检查文件格式是否正确
   - 确保文件包含必要的列（漏洞名称、风险等级、IP地址）
   - 检查文件大小是否超过限制

2. **依赖安装失败**
   - 确保Python版本为3.8或以上
   - 离线安装时确保依赖包目录存在

3. **服务启动失败**
   - 检查端口5000是否被占用
   - 检查数据库文件权限

### 日志查看

应用运行时的日志会输出到控制台，可通过查看控制台输出来排查问题。
